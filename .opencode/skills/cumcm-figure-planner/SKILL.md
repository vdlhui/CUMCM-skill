# P5c：插图方案设计

## 核心理念

**不是"论文写完了补几张图"，而是"沿着论证主线，在关键位置精准放置证据锚点"。**

每张图必须服务于一个明确的论证目的——如果删掉这张图，对应的论证会变弱。

## 输入
- P5b 纯文本论文（`final_paper_text.md`，无图）
- P5a 叙事蓝图（`narrative-blueprint.json`——核心概念、高光布局）
- P4 输出（`model_results.json` / `metrics.json` / `table_index.json`）

## 工作流

### 第 1 步：扫描全文中"值得配图"的关键段落

逐节扫描纯文本论文，标记以下类型的段落：

```
[数值对比型] → 适合数据结果图
  信号：含≥3个类别的数值对比、百分比差距、"优于"/"高于"/"降低"等比较词
  
[趋势变化型] → 适合折线图/面积图
  信号："随之增大/减小"、"呈线性/指数趋势"、"饱和"、"拐点"

[参数扫描型] → 适合敏感性分析图
  信号："当XX从A变化到B时"、灵敏度分析、"变化范围"

[结构描述型] → 适合逻辑框架图
  信号：含方法流程描述、层级/递进关系、"首先→然后→最后"

[空间关系型] → 适合热力图/三维图
  信号：含"区域分布"、"空间异质性"、"二维分布"

[统计相关性型] → 适合散点图+回归
  信号：含"相关系数"、"正/负相关"、"Spearman/Pearson"
```

### 第 2 步：对照 P5a 高光布局——强制配图

P5a 叙事蓝图中标注的 3 个高光章节，**每处必配至少一张核心图**：
- 高光 #1 → 必须配图（类型根据段落特征选择）
- 高光 #2 → 必须配图
- 高光 #3 → 必须配图

### 第 3 步：区分逻辑框架图 vs 数据结果图

| 类别 | 插入位置 | 目的 | 生成方式 |
|------|:--------:|------|----------|
| 逻辑框架图 | 问题分析后 / 模型建立章开头 | 帮评委快速看懂"我们想干什么" | Agent 生成文字描述 + logical_figure_generator.py |
| 数据结果图 | 结果分析段落紧后 | 帮评委快速验证"我们做成了什么" | figure_generator.py |

### 第 4 步：为每张图制定完整方案

每张图需回答以下问题：

```
1. 插入位置：在哪个章节/哪个句子之后？
2. 核心论证：这张图在证明什么？（一句话结论）
3. 图表类型：从 18 种类型中选择
4. 数据提取：x/y 数据来自 P4 哪个文件？哪个 JSON 路径？
5. 标注策略：图上标注什么？（最优值/差距%/显著性星号/阈值线）
6. 配色方案：使用哪套调色板？
7. 图注文本：按照黄金结构撰写（结论先行+数据支撑+论点关联）
```

### 第 5 步：输出 figure_placement_plan.json

按照 `docs/contracts/figure-placement.schema.md` 规范输出。

---

## 输出

| 文件 | 格式 | 内容 |
|------|------|------|
| `figure_placement_plan.json` | JSON | 每张图的完整方案（插入位置/论证/类型/数据/标注/配色/图注） |

## 不能做的事
- 不要为"可有可无"的段落配图——图必须是论证的必需品，不是装饰
- 不要在摘要或结论中插入图（CUMCM 惯例——摘要和结论不含图）
- 不要一张图配一句无关紧要的结论——每个图注必须包含具体数字
- 不要跳过 P4 数据直接"编造"图表数据——所有数据必须有明确追溯路径
