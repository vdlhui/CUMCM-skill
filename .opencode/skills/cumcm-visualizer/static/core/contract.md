# 图表契约（7点）

在生成任何代码之前，先回答以下 7 个问题。生成的图表必须通过全部 7 点的验证。

## 1. 核心论证
这张图在证明什么？用一句话写出结论——这句话将用作图标题。

- ❌ 不合格："不同模型的误差对比"
- ✅ 合格："国王球较高级球净收益提升193%（280898 vs 95797洛克贝/只），混合策略无法突破纯策略上限"

## 2. 图表类型
从 18 种类型中选择一种。选择原则：
- 能用柱状图说清的，不上热力图/雷达图
- 可读性永远优先于炫技

## 3. 数据契约
x 轴 / y 轴 / 误差线 / 标注值来自 P4 的哪个 JSON 文件的哪个字段？
- 必须精确到 JSON 路径
- 确保数据可追溯、可复现

## 4. 标注策略
图上直接标注哪些关键信息？
- 最优值 ✅ 标记
- 百分比差距
- 显著性星号（***/**/*/n.s.）
- 关键阈值参考线
- "takeaway" 文本框（一句话告诉评委看什么）

## 5. 面板结构
- 单面板：一张图一个论证
- 双面板（a/b）：两个子论证共享同一个问题域
- 三面板：主图+放大子图+图例面板
- 多面板的每子图必须回答不同的科学问题（反冗余原则）

## 6. 配色方案
从 5 套调色板中选择 1 套。每张图 ≤4 种颜色。

## 7. 图注（黄金结构）
图号 + 结论先行 + 关键数据支撑 + 与论文论点的关联
- 图注必须让评委在不读正文的情况下理解图的核心发现
- 结尾标注"支撑§X.X论证"
