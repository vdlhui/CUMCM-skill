# P5b：论文写作

## 角色分配
- 🟢 建模手主导：模型建立章节（技术准确性把关）、模型评价章节
- 🟢 编程手主导：求解与结果章节（算法描述+数据呈现+敏感性分析文本）
- 🟢 论文手主导：全文统稿、7 条规范自检、参考文献、概念图描述、输出前总检

## 输入
- P0 输出：problem_analysis.json（问题特征/约束/假设）
- P1 输出：model_route.json（模型链/评选依据）
- P2 输出：model-innovation.json（创新条目/公式链）
- P3 输出：model-review.json（评审结果/风险项）
- P4 输出：model-results.json / metrics.json / table-index.json / tuning_report.json
- P5a 输出：narrative-blueprint.json（核心概念/叙事主线/高光布局/摘要/章节递进结构/评分点-证据映射）
- 已注入知识库：references/winner-style-library.md

## 写作前准备

- 读取 P5a narrative-blueprint.json → 确立核心概念、叙事主线、章节递进结构
- 读取 P5a 评分点-证据映射 → 确保每个评分维度在正文中有对应证据
- 读取 P5a 高光布局 → 确保高光章节放慢节奏、加大解释密度
- 读取 winner-style-library → 国一评价风格 + 扣分句替换表 + 章节节奏分配

## 7 条评审友好规范（每节写后逐条自检，不满足→重写该节）

### 规范 ① 导航句

每节开头 1-2 句预告本节内容与结论走向。

| ✅ 通过 | 🟡 不通过 |
|--------|----------|
| "本节建立基于注意力机制的 LSTM 温度预测模型。我们先指出标准 LSTM 在本题场景下的不足，然后引入时间注意力机制进行改进，最后通过消融实验量化创新的独立贡献。" | "本节建立温度预测模型。"（没说本节要做什么、结论是什么方向） |

### 规范 ② 解释锚点

每个公式后面紧跟一句"这意味着..."的自然语言解释。

| ✅ 通过 | 🟡 不通过 |
|--------|----------|
| "式(8)中 α_t 是第 t 个时间步的注意力权重。这意味着当某个时间步的隐藏状态 h_t 与全局上下文高度相关时，模型会自动赋予其更大的权重——这正是我们需要的'自动聚焦于关键转折点'的能力。" | "式(8)定义了注意力权重。由式(8)可得..."（解释了公式是什么，没说为什么重要） |

禁用句："由式(X)可得""根据式(X)可知""代入式(X)得" → 改为"式(X)说明..."或"式(X)意味着..."

### 规范 ③ 图表自明性

图注写结论性描述，非"关系图"/"结果图"/"示意图"等空洞标题。

| ✅ 通过 | 🟡 不通过 |
|--------|----------|
| "图 3：α 取值增大导致系统从稳定振荡态→临界阻尼态过渡（α=1 为分界点），过渡区间长约 Δα=0.3" | "图 3：α 与系统稳定性关系图" |

不看正文，仅看图注+图轴，能独立理解核心结论。

### 规范 ④ 段落长度

技术段落 ≤ 150 字。超过则拆分或加小标题打断。

检查方法：对每个技术段落数字数。>150 字 → 在逻辑断点处拆分。

### 规范 ⑤ 术语一致性

P5a 的核心概念在正文中自然出现 ≥ 4 次，每次深化含义。

| 出现位置 | 作用 |
|----------|------|
| 第 3 章开头 | 引入定义——"本文提出...的核心概念..." |
| 第 4 章创新展开 | 深化——"这一概念的核心在于..." |
| 第 4 章结果分析 | 验证——"...的实验结果验证了这一概念的合理性" |
| 第 5 章结论 | 升华——"...不仅适用于本题，也为同类问题提供了...的思路" |

不是机械重复，每次都要推进对概念的理解深度。

### 规范 ⑥ 节间递进标识

每个子节/章末尾写一句过渡句。

| ✅ 通过 | 🟡 不通过 |
|--------|----------|
| "上述模型解决了单个时间点的温度预测，然而实际操作中需要在预测基础上选择最优控制参数——这涉及一个约束优化问题，我们在下一节通过将预测结果嵌入 MILP 框架来解决。" | 直接结束本节，下一节开头无任何承接 |

要求：所有节末过渡句连读（跳过正文内容），形成一段流畅的叙述。

### 规范 ⑦ 结果可视化引导

每个关键数值结果后写一句"这一结果对实际决策意味着..."。

| ✅ 通过 | 🟡 不通过 |
|--------|----------|
| "模型在测试集上 MAE=0.042（表 2），较基线 ARIMA 降低 46.2%。这一精度意味着：在焊接工艺中，温度预测误差从 ±7.8°C 降至 ±2.1°C——对于焊点质量控制而言，±2°C 的精度已满足工业标准（GB/T XXXX）的要求。" | "模型 MAE=0.042，优于基线模型的 0.078。"（没说这意味着什么） |

## 逐节写作

### 摘要（最后写，用 P5a 定稿确认）

- 使用 P5a 摘要长版（550-700 字）
- 逐项核对摘要中的数值与 P4 输出是否一致（不一致 → 必修正）
- 压缩为 300 字短版

### 问题重述与分析

- 从 P0 problem_analysis 用论文语言重述（非笔记体）
- 导航句回答"我们面对的是什么挑战，本文按什么逻辑解决"
- 至少 1 张问题结构概念图描述（不涉及模型，展示问题要素间的关系）

### 模型假设与符号说明

- 每条假设 + 一句合理性论证，不是"为简化模型"而是有数据/常识支撑
- 符号汇总表（符号-含义-单位），符号围绕核心概念命名
- 假设逐一标注相关子问题

### 模型建立与求解（论文主体，最高光章节）

**结构**：按 P5a 章节递进结构，不以子问题分节：

```
3.1 核心概念引入 + 基础模型建立
3.2 关键创新点展开（放慢节奏 → 高光 #1）
3.3 创新在 Q1 的应用与验证（消融实验 → 高光 #2）
3.4 框架拓展到 Q2（是参数调整/维度拓展，不是换模型）
3.5 框架在 Q3 的最终验证
```

每小节写完后，执行 7 条规范自检。
高光章节（P5a 标注的）刻意放缓节奏：更多导航句 + 更多解释锚点 + 更多概念图描述。

### 求解与结果

- 算法简要说明（不做完整推导，列举关键步骤）
- 结果展示（三线表格式，从 P4 table-index 引用）
- 结果分析（差异来源，不是"结果很好"）
- 敏感性分析（从 P4 tuning_report 提取）
- 每个关键数值跟一句"对实际决策意味着..."

### 模型评价

**硬性禁止句（出现即红灯，必须重写）**：

| 禁止句 | 替代要求 |
|--------|----------|
| "模型假设较为理想化" | → 具体说明哪些假设、为什么理想化、影响多大 |
| "未考虑实际中的复杂因素" | → 列出未考虑的因素、说明为什么没考虑（数据限制/时间限制）、估算若考虑会怎样 |
| "未来可进一步完善" | → 具体完善方法 + 需要什么额外条件/数据 |
| "模型具有较好的拟合效果" | → 给量化数据 + 差异分析（哪里拟合好？哪里差？为什么？） |
| "本文模型存在一定局限性" | → 从 P3+P4 提取真实张力点，量化局限的边界 |

**国一风格要求**：
- 从 P3 评审预判 + P4 实测验证中提取真实张力点
- 格式："P3 评审预判了 XX 的敏感性，P4 实测验证发现：当 α 偏离标定值 15% 时，预测误差从 2.1% 膨胀至 23%（图 X），验证了我们的预判。若引入 YY 机制（需补充 ZZ 数据），预计可将误差压缩至 8% 以内。"
- 优点有量化证据，局限有量化边界，改进有具体路径+前提条件

### 参考文献

- GB/T 7714 格式
- 正文中每条引用必须有对话：
  - "不同于文献[1]采用的 XX 方法，我们针对本题 YY 特征做了 ZZ 改造"
  - "文献[2]在类似问题上验证了 AA 假设，本题通过 BB 数据确认该假设成立"
  - "文献[3]提出的 CC 框架启发了我们的 DD 设计"
- 不是末尾堆清单

### 附录

- P4 核心代码摘要（关键算法逻辑）

## 逐节写作后：微单元覆盖性检查

写完每节后，加载 `cumcm-micro-units` Skill，对照该节的句子级清单逐项检查：
- ✅ 该句在正文中有对应内容（意思到了即可）
- ⚠️ 该句内容正文中缺失 → 补写
- — 该句不适用本题 → 跳过

此步骤确保"该写的句子都写了"，不是模板填空。

## 输出前总检

| 检查项 | 判据 | 检查方法 |
|--------|------|----------|
| 核心概念贯穿 | 全文搜索核心概念 → 出现 ≥ 4 次，每次深化含义 | 搜索工具统计 |
| 导航句密度 | 每节有导航句，所有导航句连读成小摘要 | 抽取导航句连读测试 |
| 节间逻辑钩子 | 所有节末过渡句连读成流畅叙述 | 抽取过渡句连读测试 |
| 图表独立可读 | 只看所有图注（不读正文），能否理解论文做了什么 | 抽取图注连读测试 |
| 反模板句清零 | 全文搜索禁止句列表 → 0 命中 | 搜索工具扫描 |
| 摘要数值一致 | 摘要中数字与 P4 输出逐个核对 | 人工比对 |
| 评分点覆盖 | 对照 P5a 评分点-证据映射，每个维度在正文中有对证 | 逐项核对映射表 |

任一项不通过 → 标注问题返回对应节重写。

## 输出

论文正文稿（Markdown 格式，含全部章节内容、公式、图表描述、引用、附录），可直接灌入 LaTeX 模板。

## 不能做的事
- 不要写"子问题 1→子问题 2→子问题 3"的并列结构（P5a 已设计了递进结构）
- 不要让模型评价中出现任何一条禁止句（硬性红灯，无例外）
- 不要在正文中裸奔数据（每个数值必须有单位/对比基准/公式引用）
- 不要在参考文献中堆砌而不在正文中对话
- 不要跳过 7 条规范自检——每节写完后逐条检查，不通过就重写该节
