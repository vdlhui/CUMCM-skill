# P5a：叙事设计

## 角色分配
- 🟢 建模手主导：核心概念确认、高光技术点筛选和论证
- 🟡 编程手辅助：确认高光章节的数据支撑是否到位
- 🟢 论文手主导：叙事主线、摘要漏斗、正文结构图、评分点映射、自评审

## 输入
- P0 输出：problem_analysis.json（问题特征/套路匹配）
- P1 输出：model_route.json（模型链/三维评分）
- P2 输出：model-innovation.json（创新条目/公式链）
- P3 输出：model-review.json（评审结果/风险项）
- P4 输出：model-results.json / metrics.json / table-index.json（数值结果素材）
- 已注入知识库：
  - references/cumcm-grading-guide.md（评分维度）
  - references/winner-style-library.md（摘要模板/创新呈现模式/章节节奏/国一评价风格/扣分句替换）

## 执行步骤

### 第 1 步：选择摘要模板

根据 P0 的套路匹配和题目特征，从 winner-style-library 中选择摘要模板：

- 物理/工程类 → 模板 A：问题穿透型
- 大数据/ML 类 → 模板 B：数据驱动型
- 评价/决策类 → 模板 C：指标驱动型

### 第 2 步：选定创新呈现模式

从 winner-style-library 选择创新呈现模式：

- P2 创新来自"标准模型缺陷" → 模式 A：缺陷驱动型
- P2 创新来自"数据/物理规律洞察" → 模式 B：洞察驱动型

### 第 3 步：提炼核心概念

用一个短语（3-8 个字）概括全文的核心创新思想。要求：
- 贯穿所有子问题（不是某个子问题特有的）
- 体现创新本质（不是模型名称的堆砌）
- 评委看完能记住

**示例**：
- "动态耦合温度预测网络"（而非 "LSTM+注意力机制"）
- "分层博弈决策框架"（而非 "博弈论+TOPSIS"）
- "反演-正演闭环验证体系"（而非 "有限差分+最小二乘"）

### 第 4 步：设计叙事主线（150 字）

说明三个子问题如何围绕核心概念展开。要求：
- 不是罗列子问题（"问题一用了 A，问题二用了 B，问题三用了 C"）
- 而是说明递进/并行/反馈关系
- 每个子问题在本框架下的定位

**示例**（递进型）：
> "单一预测模型无法同时捕捉温度场的趋势与突变→引入注意力机制解耦趋势/突变两个尺度→通过耦合损失函数联合优化→三个子问题在主框架下逐层递进：Q1 建立基础温度预测，Q2 在预测基础上引入工艺约束优化控制参数，Q3 将优化结果回代验证预测的鲁棒性。"

### 第 5 步：高光布局

选出 3 个评委应停留阅读的段落/公式/图表，说明为什么：

| # | 高光项 | 所在位置 | 打动力理由 |
|---|--------|:------:|----------|
| 1 | [公式/图表/段落] | [预计所在章节] | [为什么评委应该在这里停留] |

**示例**：
- "注意力权重可视化（式 8-9 的热力图）：直观展示模型抓住了题目中的关键转折点——这是评委能秒懂的创新证据"
- "耦合损失函数对比实验（表 3）：量化说明为何联合优化优于分开训练——Ablation Study 是国一标配"
- "P3 预判 vs P4 实测对比（图 9）：展示了对自己模型局限性的真实认知——真诚的自我解剖得分远超万能模板"

### 第 6 步：撰写摘要漏斗

**长版（550-700 字）**：

按选中模板的结构写作，每部分检查：

```
钩子(问题本质+为什么难)   [是否直击问题核心？是否说明了难度？]
↓
核心洞察(用什么视角/工具破解) [是否体现创新？是否与其他方案区分？]
↓
模型骨架(逻辑关系，不是清单) [是否体现链结构？是否有递进感？]
↓
关键结果(有对比基准的数字)   [是否有具体数值？是否有基线对比？]
↓
结论延伸价值                  [是否有超出赛题的启发？是否"说明"了什么？]
```

**短版（300 字）**：从长版压缩，保留钩子+核心洞察+最关键的一个结果+结论。删减推导细节和次要数据。

### 第 7 步：正文递进结构图

设计围绕核心概念的主线推进结构，不是"子问题 1→子问题 2→子问题 3"的并列结构：

```
第三章：核心概念引入 → 基础模型建立
  ↓
第四章：关键创新点展开 → 子问题递进应用
  - 4.1：基础模型在 Q1 的应用（快速验证框架可行性）
  - 4.2：创新引入后 Q1 效果的提升（量化创新贡献）
  - 4.3：将框架拓展到 Q2（不是"换模型"，是"调整参数/拓展维度"）
  - 4.4：框架在 Q3 的最终验证
  ↓
第五章：模型评价（P3 预判 vs P4 实测的真实张力）
```

### 第 8 步：评分点-证据映射

基于 cumcm-grading-guide 评分维度，建立映射表：

| 评分维度 | 对应章节 | 证据来源 | 证据内容 |
|----------|:------:|------|------|
| 模型创新性 | 4.2 | P2 model-innovation.json | 注意力机制的独立贡献通过消融实验确认 |
| 求解正确性 | 4.3 | P4 table-index.json | 表 3 展示与基线 ARIMA 的对比 |
| 模型真实性 | 5 | P3 model-review.json + P4 稳定性 | 评审预判的敏感性在实测中确认 |

### 第 9 步：自评审

| 检查项 | 判据 | 不通过动作 |
|--------|------|-----------|
| 核心概念贯穿 | 核心概念能回溯到每个子问题的建模方案 | 重新提炼核心概念 |
| 叙事递进 | 阅读章节标题就能感受到"由浅入深" | 调整章节结构 |
| 摘要独立可读 | 摘要不依赖正文即可理解"做了什么/怎么做的/做到了什么程度" | 重写摘要 |
| 摘要模板匹配 | 调用了 winner-style-library 对应模板 | 检查模板选择是否正确 |
| 创新呈现一致 | 创新呈现模式（缺陷驱动/洞察驱动）全文一致 | 统一创新呈现风格 |
| 反模板句检查 | 摘要和结构图中无扣分原句（见 winner-style-library 第五章） | 替换为具体描述 |

若任一检查不通过 → 重做对应步骤。

## 输出

| 文件 | 格式 | 内容 |
|------|------|------|
| `narrative-blueprint.json` | JSON | 核心概念/叙事主线/高光布局(3个)/摘要长版(550-700字)/摘要短版(300字)/章节递进结构/评分点-证据映射表 |

## 不能做的事
- 不要用"子问题 1→子问题 2→子问题 3"的并列结构（那是技术报告，不是国一论文）
- 不要在摘要中罗列模型名而没有逻辑关系（"本文使用了 AHP、TOPSIS 和 LSTM" → 零分摘要）
- 不要在摘要中出现"本文""我们"以外的人称
- 不要在摘要中出现无对比基准的裸数字
- 不要在高光布局中选"完整性"类项目（如"参考文献格式正确"）——高光必须是创新/验证/结果类的炸点
