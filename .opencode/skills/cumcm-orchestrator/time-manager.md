# CUMCM 时间管理器

## 一、总预算

| 项目 | 说明 |
|------|------|
| 总赛时 | 72 小时（通常周四 20:00 发布 → 周日 20:00 提交） |
| 有效建模时间 | 约 48~55 小时（扣除吃饭、睡觉、排版） |
| 最晚提交前预留 | 至少 8 小时用于排版 + 最后检查 |

---

## 二、阶段时间预算

| 阶段 | 标准预算 | 最晚累计 | 超时动作 |
|------|:------:|:------:|------|
| **P0** 读题 | 2h | 2h | 必须完成才能启动 P1 |
| **P1** 选模 | 3h | 5h | 超过 3h → 强制收敛候选数 ≤ 3 |
| **P2** 创新 | 4h | 9h | 超过 4h → 限制创新条目 ≤ 2 |
| **P3** 评审 | 1h | 10h | 回环上限 2 次，第 3 次直接降级放行 |
| **P4** 求解 | 12h | 22h | 超过 10h → 禁止新增求解脚本，专注调参 |
| **P5a** 叙事 | 2h | 24h | — |
| **P5b** 写作 | 8h | 32h | — |
| **排版+修改** | 10h | 提交前 | — |

---

## 三、四模式自动降级

### 完整模式（剩余 > 36h）

**触发**：赛时前 36 小时，或进度前于预算

**允许**：
- ✅ P4 允许深度学习模型（GPU 训练 > 30min 允许）
- ✅ P4 允许大规模仿真（蒙特卡洛 M > 10⁵ 次允许）
- ✅ P1 允许 ≥ 5 候选全面对比
- ✅ P3 允许回环 3 次
- ✅ P2 允许创新 ≥ 3 条
- ✅ 允许从头训练模型

---

### 效率模式（剩余 24-36h）

**触发**：赛时进入第 2 天，或 P4 超预算

**约束**：
- ⚠️ P4 禁止从头训练深度学习模型 → 只允许预训练/轻量模型（统计/经典ML）
- ⚠️ P4 限制仿真次数 M ≤ 10⁴
- ⚠️ P1 强制收敛候选数 ≤ 3
- ⚠️ P3 回环上限 1 次
- ⚠️ P2 创新上限 2 条
- ⚠️ 禁止切换模型大类（只能在同一类内换具体模型）

---

### 保守模式（剩余 12-24h）

**触发**：赛时进入第 3 天，或 P4 严重超预算

**约束**：
- 🔒 P4 只允许线性模型/统计模型/经典 ML（随机森林/XGBoost）
- 🔒 P4 禁止仿真
- 🔒 P1 锁定已选模型链，禁止重新选型
- 🔒 P3 禁止回环（仅标注风险直接放行，黄灯 → 绿灯）
- 🔒 P2 禁止创新（使用标准模型即可，已有创新保留但不新增）
- 🔒 P5a 允许跳过叙事设计（使用通用模板）

---

### 写作优先模式（剩余 < 12h）

**触发**：距离提交不足 12 小时

**约束**：
- 🔒 P4 只允许整理已有结果，禁止运行新代码
- 🔒 跳过 P5a，直接使用摘要通用模板（从 winner-style-library 匹配）
- 🔒 全部精力 → P5b 写作 + 排版
- 🔒 所有 ⚠️/🟡/🔴 → ✅ 放行
- 🔒 禁止任何回环
- 🔒 禁止修改模型/代码/数据

---

## 四、超时检测机制

```
每次阶段推进前：
  1. 计算已用总时间
  2. 确定当前时间模式
  3. 若模式降级 → 向 Agent 注入降级模式说明和约束清单
  4. 更新 GCD 时间追踪行
  5. 若进入写作优先模式 → 注入"停止一切建模工作，全力写作"的强制指令
```
