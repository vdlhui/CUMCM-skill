## 一、主流程状态转换

```
[赛题输入] 
    │
    ▼
  启动(初始化GCD→注入P0知识库)
    │
    ▼
┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐  ┌───────────┐
│  P0   │→│  P1   │→│  P2   │→│  P3   │→│  P4   │→│  P5a  │→│  P5b  │→│  P5c  │→│Visualizer│→[定稿]
│ 读题  │  │ 选模  │  │ 创新  │  │ 评审  │  │ 求解  │  │ 叙事  │  │纯文本│  │插图方案│  │  图表生成  │
└───────┘  └───────┘  └───────┘  └───────┘  └───────┘  └───────┘  └───────┘  └───────┘  └───────────┘
    │           ▲           ▲        ▲  │        │
    │           │           │        │  │        │
    │           └───方向性回环──┘  修正 │        │
    │                                回环       │                       │
    │                                    │      │                       │
    └────────── 否决回退 ────────────────┘      │                       │
                                                 │                       │
                                    ┌────────────┼───────────┐           │
                                    │            │           │           │
                               结构性→P2   可解性→P3   数值→P4内循环     │
                                                                        │
                                P5a自评审不通过→重写 P5a                 │
                                P5b总检不通过→标注返回P5b重写           │
```

## 二、P3 评审回环路由

```
P3 评审完成
    │
    ├─ 综合判定：绿灯（全维绿灯） 
    │   └─ → P4 求解
    │
    ├─ 综合判定：黄灯 ≤ 1 
    │   └─ → P4 求解（携带风险标注列表）
    │
    ├─ 综合判定：黄灯 ≥ 2 
    │   └─ → 返回 P2 修正（携带：哪些维度黄灯 + 修正建议）
    │
    ├─ 综合判定：红灯（方向再审）
    │   └─ → 返回 P1 重新选模（携带：驳回原因 + 竞争模型的优势证据）
    │
    └─ 综合判定：红灯（创新可行性/完整覆盖性）
        └─ → 返回 P2 修正创新（携带：推导漏洞/未覆盖子问题的具体说明）
```

### 回环携带信息格式

```markdown
## 回环指令
- 来源阶段：P3
- 目标阶段：P1
- 回环类型：方向性回环
- 驳回原因：竞争模型[模型名]在[指标]上比当前方案优[X%]，且假设更弱
- 需重新回答的问题：
  1. [具体问题]
  2. [...]
- 携带的上下文：[被驳回的 model-route.json 摘要]
```

## 三、P4 失败诊断三分支回环

```
P4 求解
    │
    ├─ 求解成功 
    │   ├─ 结果验证通过 → 输出素材包 → P5a
    │   └─ 结果不达标 → auto-tuner 调参
    │       ├─ 调参后达标 → 输出
    │       └─ 调参后仍不达标 → model-validator 否决
    │           ├─ 方向性错误 → 回退 P1 备选链
    │           ├─ 创新引入问题 → 回退 P2 移除该创新
    │           └─ 数据不支持 → 回退 P0 检查数据质量/外部数据
    │
    └─ 求解失败（崩溃/无解/异常）
        ├─ 结构性问题（无解/矛盾/物理意义不符）
        │   └─ → P2 调整模型结构
        ├─ 可解性漏判（内存溢出/时间爆炸/完全无法收敛）
        │   └─ → P3 重新评审可解性（标记上次漏判盲区）
        └─ 数值不稳定（初值敏感/震荡发散/步长失效）
            └─ → P4 内循环（换算法/求解器/初始策略/精度设置）
```

## 四、ABCD 四级降级策略

| 判定等级 | 含义 | 动作 | 剩余时间条件 |
|:--------:|------|------|:-----------:|
| **A 通过** | 全部绿灯 | 放行下一阶段 | 任何时间 |
| **B 局部修正** | ≤1 黄灯 | 放行 + 标注风险 | 任何时间 |
| **C 结构重调** | ≥2 黄灯 | 默认返回修正 | 剩余 >12h 时必须返回修正 |
| | | 剩余 <12h 时可降级放行 | 剩余 <12h |
| **D 推倒重来** | 任一红灯 | 必须回环指定阶段 | 任何时间（不可降级） |

## 五、时间约束触发规则

```
每次阶段推进前，Orchestrator 检查 time-manager：

1. 计算剩余时间
2. 确定当前模式（完整/效率/保守/写作优先）
3. 检查当前阶段是否超过该模式的预算
4. 若超预算：
   - 触发降级动作（见 time-manager.md 各模式约束）
   - 更新 GCD 当前模式
   - 向 Agent 注入降级提示
```

## 六、全局禁止规则

| 规则 | 说明 |
|------|------|
| 禁止跳阶段 | 任何情况下必须按 P0→P1→P2→P3→P4→P5a→P5b 顺序，不可跳过 |
| 禁止 P3 绕过 | 即使时间紧迫，P3 红灯必须回环，不可直接进 P4（红灯=D 级，不可降级） |
| 禁止超过回环上限 | 同阶段同原因回环 ≤2 次，第 3 次自动降级（若可降级）或报错（若不可降级） |
| 禁止空输出 | 每阶段结束必须产生符合契约的输出，状态标记根据实际产出决定 |
