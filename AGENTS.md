# CUMCM-skill — 数学建模竞赛论文生成系统

## 触发规则（最重要，请先读这个）

当用户发送以下任一类型的消息时，**立即自动加载 cumcm-orchestrator**：

- 直接粘贴赛题原文（长文本、含数字/公式/表格）
- "帮我做这道数学建模题" / "帮我建模" / "做这道题"
- "开始建模" / "分析这道题" / "解这道竞赛题"
- "启动" / "开始" / "建模" + 后面跟着任何疑似赛题的内容
- 上传 PDF / 图片 / Word 格式的赛题文件
- `/cumcm` 及其任何变体（见斜杠命令）

**简单来说：用户不需要用任何 @ 前缀或精确命令名。他们只要说"我要建模"，或直接扔赛题过来，就该自动启动。Orchestrator 加载后会识别赛题内容、初始化 GCD 并按阶段推进。**

## 怎么启动

```
# 最简单的方式 —— 直接说话
帮我做这道题：[粘贴赛题]
开始建模：problem_files/2024A.pdf

# 斜杠命令（快捷方式）
/cumcm [赛题]             交互式，逐阶段审阅推进
/cumcm auto [赛题]         自动化，一键执行全流程
/cumcm status               查看当前进度
/cumcm resume               从断点恢复

# 脚本命令（调试用）
python scripts/run_pipeline.py --problem "赛题文件路径"
```

## 七阶段流水线

```
[赛题] → P0 读题 → P1 选模 → P2 创新 → P3 评审 → P4 求解 → P5a 叙事 → P5b 写作 → [论文稿]
```

| 阶段 | 名称 | 核心职责 |
|:--:|------|------|
| P0 | 读题与数据处理 | 赛题解析、子问题分解、套路识别、数据清洗画像 |
| P1 | 模型方向选型 | 模型链搜索、三维评分（得分×成本×风险）、路线决策 |
| P2 | 模型创新优化 | 创新模式匹配、公式链推导、可行性评估 |
| P3 | 模型评审闸门 | 五维量化评审、数学验证、回环路由判定 |
| P4 | 求解与代码 | 代码生成、自动调参、模型否决、失败诊断、证据门禁 |
| P5a | 叙事设计 | 核心概念、叙事主线、高光布局、摘要漏斗 |
| P5b | 论文写作 | 7 条规范逐节写作、微单元覆盖检查、反模板句检查、格式门禁 |

## 关键机制

- **回环**：P3 评审不通过 → 精确回退 P1/P2；P4 求解失败 → 三分支诊断回环
- **时间管理**：四模式自动降级（完整>36h / 效率24-36h / 保守12-24h / 写作优先<12h）
- **角色协调**：三人角色（建模手/编程手/论文手）每阶段任务分配
- **GCD**：全局上下文文档，六维视图（阶段状态/证据状态/风险/回环/时间/角色）
- **双门禁**：P4 证据门禁（evidence_status 必须 complete）+ P5b 格式门禁（字数/标题/引用/反模板句）
